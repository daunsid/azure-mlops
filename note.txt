azure ml components

ML Workspace: Logical abstraction of all ML asssets like compute, blobs, endpoints
Compute Target: Virtual Machine used for training/dev
Data set: Metadate and refs to data at diff srcs like blob, data lakes etc
experiment: Exp is a logical groping of all the trails or runs
Run: Each run can log the metrics, images, outputs, code snapshots, explanations & fairness
model registry: manage the models with proper versions and lineage
model profilling: identify memory & processing req for a model
model deployment: single/multi-model deployment in azure container instances/kubernetes
real time data capture: store the real time input & prediction from the model
data drifts: identify and take appropraite actions with the changes in features, distributions, data quality, natural data shift, chage in relationships btw features
